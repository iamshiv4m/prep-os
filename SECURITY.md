# Security Policy

## Supported Versions

Only the most recent release of PrepOS receives security fixes. We're early
in v0.x — there is no LTS line yet.

| Version  | Supported          |
| -------- | ------------------ |
| latest   | :white_check_mark: |
| < latest | :x:                |

## Reporting a Vulnerability

If you discover a security issue in PrepOS, **please don't open a public
GitHub issue.** That tells everyone — including the next attacker — about
the bug before there's a fix.

Instead, report it privately:

1. **GitHub private advisory** (preferred): go to the repo's
   **Security → Advisories → "Report a vulnerability"** button.
2. **Email**: `hello@prepos.app`. PGP key on request.

Please include:

- A description of the issue and its impact.
- Steps to reproduce, ideally with a minimal proof-of-concept.
- Affected version(s) (`PrepOS → Menubar → About` shows the running version).
- Your OS and Electron version, if you know it.

### What to expect

- We aim to acknowledge reports within **48 hours**.
- A fix usually ships in the next release (~1 week for high severity).
- We'll credit you in the changelog and the GitHub advisory unless you ask
  to stay anonymous.

Thank you for helping keep PrepOS users safe.
